What is this?

LabScope searches across the accreditation schedules of 1,524 UKAS-accredited testing and calibration laboratories. These schedules define exactly what each lab is accredited to test, including the materials, test methods, and standards covered.

Until now, this data was locked inside individual PDF documents on the UKAS website. Finding a lab for a specific test meant opening schedules one by one, or relying on word of mouth. LabScope makes all 14,298 capabilities searchable in one place.

Search uses a hybrid approach: full-text keyword matching (so "BS 4449" finds labs accredited to that standard) combined with semantic vector search (so "will this weld hold?" finds tensile and fatigue testing labs). Results are ranked using Reciprocal Rank Fusion to give you the best of both.

API & AI Agent Access

LabScope exposes a REST API and an MCP (Model Context Protocol) server, so AI agents and applications can search lab capabilities programmatically.

REST API
GET /api/search?q=tensile+testing&limit=10&region=London
POST /api/match — find labs covering multiple test requirements
GET /api/labs/{lab_id} — full lab details + all capabilities
MCP Server

Connect your AI agent to LabScope via the Model Context Protocol. The MCP endpoint is available at /mcp and provides three tools:

search_lab_capabilities(query, limit, region)
get_lab(lab_id)
find_labs_for_multiple_tests(queries[], limit, region)
Data source: All capability data is extracted from publicly available UKAS (United Kingdom Accreditation Service) accreditation schedules. UKAS is appointed by the UK government as the national accreditation body. Lab accreditation status should always be verified against the official UKAS directory at ukas.com before making procurement decisions.
